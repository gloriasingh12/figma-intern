# Task 6: Collaborative Design Workflow in Figma
**Project:** Shared Branding & UI Design for TechAditya
**Objective:** Managing team feedback, versioning, and real-time collaboration.

---

## 1. Project Setup & Sharing
* **File Structure:** Created a dedicated 'Design System' and 'Mockups' page.
* **Access Control:** Shared the project with team members (Stakeholders and Devs) using "Can Edit" and "Can View" permissions via email invites.

## 2. Real-time Commenting & Feedback Loop
* **Communication:** Used the 'Comment' tool (shortcut: C) to tag specific team members on design elements.
* **Contextual Feedback:** * *Example Comment:* "@Aditya, can we increase the contrast on the 'Call to Action' button for better accessibility?"
* **Resolution:** Once the design was updated, comments were marked as 'Resolved' to maintain a clean workspace.

## 3. Version Control & History
* **Checkpointing:** Created manual versions (File > Save to Version History) before making major changes.
* **Audit Trail:**
    * *Version 1.0:* Initial wireframes.
    * *Version 1.1:* Added Color Palette & Typography.
    * *Version 2.0:* Final Mockups after team review.
* **Recovery:** Demonstrated the ability to 'Restore' an older version in case a design direction was pivoted.

## 4. Handoff to Developers
* **Inspect Mode:** Enabled the 'Dev Mode' (or Inspect Panel) to provide CSS, iOS, and Android code snippets to the engineering team.
* **Asset Export:** Prepared icons and logos in SVG/PNG formats for direct use in the frontend.
